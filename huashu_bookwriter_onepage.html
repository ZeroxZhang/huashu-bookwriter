创作工作流
01

确定蓝图与定位

选择适用的书籍 Type (A/B/C)，明确目标读者群与整体信息密度期望。

02

设计大纲结构

依据蓝图定义 Part 结构与章节编号，生成标准的 outline-schema。

03

信息源搜集

从官方文档、社区资源及源码中提取硬核知识，确立技术正确性。

04

模块化撰写

采用概念讲解、实战教程或深度分析模板，并严格注入“花叔风格 DNA”。

05

双重质量校验

执行 12 项章节级 QC 与 10 项全书级 QC，确保编号连续、格式统一。

06

编译与发布

一键通过 Pandoc + XeLaTeX 引擎导出具备精美排版的高清 PDF。

三大书籍蓝图

Type A · 经典技术手册

框架 / 标准

结构设定 10章±2，划分 3 个 Part，使用中英双语标题

核心特征 结构严谨，知识点全覆盖，适合作为官方文档转化或系统性框架的权威指南。

Type B · 深度技术文档

源码 / 底层

结构设定 17-35节，5+ 个 Part，极高密度的代码块展示

核心特征 聚焦于源码级解析、底层逻辑剖析与架构推演，面向资深开发者的硬核读物。

Type C · 快速实战指南

入门 / 实践

结构设定 12-17章，采用“核心概念 + 实战案例”交替模式

核心特征 强调快速上手与最佳实践落地，大幅降低阅读门槛，适合新技术尝鲜与培训。

🧬
写作风格 DNA
  • 极简句式约束：单句长度严格控制在 25 字以内，增强阅读节奏感。
  • 对话式代入感：高频使用第一人称（"我"、"我们"），打破技术说明文的冰冷感。
  • 量化与具象化：使用具体数据与明确定义，拒绝使用模糊量词。
  • AI 味消除：系统级禁用"综上所述"、"值得注意的是"等常见机器生成套话。
✅
质量控制系统

章节级检查 (12项)

标题非空校验、Markdown 语法规范检查、代码块闭合完整性、警告/提示区块模式验证。

全书级检查 (10项)

专业术语全局统一性扫描、交叉引用链接有效性、Part 结构完整度、全局风格基调一致性评估。
