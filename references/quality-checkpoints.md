# 质量检查清单

每章和全书完成后必须通过的验证清单。

---

## 每章 QC（12 项）

### 结构检查（4 项）

- [ ] **§NN 格式**：章节标题用 `## §01` 格式（§ 符号 + 两位数字）
- [ ] **断言句标题**：章节标题是完整断言句，不是主题词
  - ✅ `## §01 Claude Code 改变了什么`
  - ❌ `## §01 概述`
- [ ] **时间线开头**：每章前 2-3 段有具体时间线锚点
  - ✅ "用了 3 个月 Cursor 之后切换到 Claude Code"
  - ❌ "很多用户反映..."
- [ ] **向前桥接**：章节结尾有过渡句拉读者进下一章
  - ✅ "装完了，账号登了。下一章，开始做真实项目。"
  - ❌ "以上就是本章全部内容。"

### 风格检查（5 项）

- [ ] **无禁用词**：见 style-dna.md 禁用词表
  - 无"综上所述"、"值得注意的是"、"接下来我们将"、"进行操作"
- [ ] **"我"频率**：随机抽查 200 字内至少出现 1 次"我"
- [ ] **句长控制**：随机抽查 10 句，单句 ≤25 字
- [ ] **具体数字**：用"3 个月"而非"很长一段时间"
- [ ] **产品全名**：用"Claude Code"而非"某个 AI 工具"

### 内容检查（3 项）

- [ ] **代码语言标签**：所有代码块带语言标签
- [ ] **行内注释**：关键代码行带行内注释
- [ ] **表格解读**：每个对比表格后有"花叔的结论"列 + 解读段落

### 内容块频率

| 文章类型 | 经验框 | 建议框 | 注意框 |
|---------|--------|--------|--------|
| 概念讲解 | 每章 1-2 个 | 每章 1 个 | 按需 |
| 实战教程 | 每章 1-2 个 | 关键步骤 | 每个坑 |
| 深度分析 | 每章 1 个 | 每章 1 个 | 按需 |

---

## 全书 QC（10 项）

### 全局结构（5 项）

- [ ] **元数据块完整**：创建者、为谁创建、基于、最后更新、适用场景
- [ ] **Part 分组**：有逻辑分组的 Part
- [ ] **编号连续**：章节编号从 §01 开始连续递增，无跳号
- [ ] **TOC 兼容**：所有章节用统一 `## §NN` 格式
- [ ] **阅读指南**：书末有按天/按主题的阅读建议（从入门到精通类型必须有）

### 一致性（3 项）

- [ ] **开头模式一致**：每章前 2-3 段均用时间线锚点/场景还原
- [ ] **内容块格式一致**：经验框/建议框/警告框格式全书统一
- [ ] **表格格式一致**：对比表格必须有"花叔的结论"列

### PDF 就绪（2 项）

- [ ] **第一个 H1 是标题**：用于封面生成
- [ ] **无脏 Markdown**：无未闭合的标签、无断裂的表格

---

## QC 输出格式

完成检查后，输出报告：

```markdown
## 质检报告

### 每章 QC（§XX）
**结构检查**：✅/❌ [具体问题]
**风格检查**：✅/❌ [具体问题]
**内容检查**：✅/❌ [具体问题]

### 全书 QC
**全局结构**：✅/❌
**一致性**：✅/❌
**PDF就绪**：✅/❌

### 需修复项
1. [问题描述]
2. [问题描述]
```

## TDD 铁律

**违反以下任何一条 = 删除重写**

- 开头有时间线锚点或场景还原
- 章节标题是断言句
- 至少 1 个经验框
- 对比表格有"花叔的结论"列
- 代码块有语言标签
- 无禁用词
- 结尾有向前桥接
- "我"出现频率 ≥ 每千字 3 次
- 句长 ≤25 字

---

## Rationalization Table

Agent 在压力下会找借口跳过规则：

| 借口 | 现实 |
|------|------|
| "开头不重要，内容才是关键" | 花叔风格的核心就是开头。没有锚点的章节不可信。 |
| "用'综上所述'更正式" | 花叔的书不正式。口语化是刻意选择。 |
| "这个表格不需要'花叔的结论'列" | 没有个人观点的表格是参数表，不是评测。 |
| "代码不需要注释" | 花叔的书是手把手教学。 |
| "不用向前桥接，下一章会自然衔接" | 不会。读者会在这里停下来。 |
| "句子确实有点长，但意思表达得很清楚" | 花叔的风格就是短句。 |
| "这章没有个人经历素材可用" | 没体验就诚实标注，不是跳过。 |
| "赶时间，跳过内容块" | 没有经验框的章节是说明书。 |

## Red Flags

看到以下信号，停止写作，回到章节开头重来：

- 开头是"在当今这个...时代"、"随着...的发展"、"近年来..."
- 用了"综上所述"、"值得注意的是"、"接下来我们将"
- 单句超过 25 字（逗号串多个分句）
- 表格没有"花叔的结论"列
- 代码块没有语言标签
- 章节结尾是"以上就是本章内容"、"感谢阅读"
- 全章没有"我"或"我的"
- 没有经验框/建议框/警告框
- 用"某个 AI 工具"、"这类产品"等泛称
