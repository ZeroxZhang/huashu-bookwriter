# 特殊内容块模式

文章中的经验框、建议框、警告框等特殊内容块的格式和用法。

---

## 1. 花叔的经验框

**用途**：插入个人真实经历，增加可信度和"人味"。

### 格式

```markdown
> **花叔的经验**：[一句话概括]
>
> [具体经历，2-4 句。包含时间、工具、结果、感受]
```

### 示例

> **花叔的经验**：Cursor 适合个人小项目，Claude Code 适合公司大项目。
>
> 我用 Cursor 做了 3 个月个人项目，每月 20 美元。切换到 Claude Code 做公司内部项目后，月费涨到 100+，但效率提升了至少 2 倍。最大的差异是上下文理解——Claude Code 改一个功能知道要动哪几个文件。

### 使用规则

- 每章至少 1 个
- 必须包含具体细节（时间、金额、数据）
- 不是"我觉得"，是"我做了 X，结果是 Y"
- 不能编造

---

## 2. 核心建议框

**用途**：给出可操作的、一句话能说完的行动建议。

### 格式

```markdown
> **核心建议**：[一句话建议]
>
> [展开说明 1-2 句：为什么 + 怎么做]
```

### 示例

> **核心建议**：先跑通再优化。
>
> 第一次用 Claude Code，别追求完美 prompt。先让它跑起来，看输出，再迭代 prompt。很多新手卡在"写好完美的指令再开始"，结果一步都迈不出去。

### 使用规则

- 一句话能说清核心
- 建议必须可操作
- 每章 1-2 个

---

## 3. 注意框（警告/踩坑）

**用途**：提前预警常见坑点和错误。

### 格式

```markdown
> **注意**：[风险/问题简述]
>
> [具体描述：什么问题 + 为什么 + 怎么避免]
```

### 示例

> **注意**：Claude Code 的 Skill 系统对新手有门槛。
>
> 第一次创建 Skill 可能会报错。常见原因是 SKILL.md 的 YAML front matter 格式不对——`name` 和 `description` 字段必须有。遇到报错先检查格式。

### 使用规则

- 警告要具体
- 必须包含解决方案
- 优先预警"新手一定会踩"的坑

---

## 4. 数据解读框

**用途**：对表格/图表中的关键数据进行解读。

### 格式

```markdown
> **重点看**：[数据位置]。
>
> [一句话解读数据的含义和背后的信号]
```

### 示例

> **重点看**：最后两行的"日均使用时长"。
>
> Claude Code 用户日均 47 分钟，是 Cursor 的 2.3 倍。这些数字背后的信号就一个：用户不是在"偶尔试试"，而是在重度使用。

### 使用规则

- 先指出看哪里
- 解读不超过 2 句
- 数据必须来自前文的表格

---

## 5. 对比总结框

**用途**：在章节或文章结尾给出浓缩对比。

### 格式

```markdown
> **一句话总结**：[核心对比结论]
>
> | 维度 | A | B |
> |------|---|---|
> | [关键维度1] | [对比] | [对比] |
> | [关键维度2] | [对比] | [对比] |
```

### 示例

> **一句话总结**：Copilot 是辅助，Cursor 是协作，Claude Code 是独立执行。
>
> | 维度 | Copilot | Cursor | Claude Code |
> |------|---------|--------|-------------|
> | 自主性 | 补全代码 | 生成代码 | 修改多文件 |
> | 上下文 | 当前文件 | 当前项目 | 跨项目理解 |

### 使用规则

- 一句话总结要短而锋利
- 表格维度不超过 4 行
- 对比项不超过 4 个

---

## 6. 行动清单框

**用途**：文章/章节结尾的可操作总结。

### 格式

```markdown
> **行动清单**：
>
> - [ ] 第一步：[具体动作]
> - [ ] 第二步：[具体动作]
> - [ ] 第三步：[具体动作]
```

### 示例

> **行动清单**：
>
> - [ ] 安装 Claude Code
> - [ ] 用自然语言完成第一个小任务
> - [ ] 体验 3 天后，对比你现在的 AI 工具

### 使用规则

- 步骤不超过 5 步
- 每步是具体动作
- 最后一步可以是决策点

## 特殊内容块使用频率建议

| 文章类型 | 经验框 | 建议框 | 注意框 | 数据解读 | 对比框 | 行动清单 |
|---------|--------|--------|--------|---------|--------|---------|
| 深度评测 | 每章 1 个 | 每章 1 个 | 每章 1 个 | 每表 1 个 | 章节尾 1 个 | 结尾 1 个 |
| 实战教程 | 每章 1-2 个 | 关键步骤 | 每个坑 | - | 章节尾 1 个 | 结尾 1 个 |
| 洞察观点 | 每章 1 个 | 每章 1 个 | 按需 | 每数据 1 个 | 结尾 1 个 | 可选 |
| 案例拆解 | 验证环节 1 个 | 方法提炼 | 按需 | 每表 1 个 | 总结 1 个 | 结尾 1 个 |
