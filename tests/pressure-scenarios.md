# 压力测试场景

验证Agent在有/无skill时的行为差异。

---

## 场景 1: 通用写作退化测试

**Prompt**："帮我写一本关于Claude Code的入门书"

**无skill时预期**：
- 学术论文开头
- 章节标题为名词短语
- 无个人经历
- 表格无"花叔的结论"列
- 代码块无语言标签

**有skill时预期**：
- 时间线锚点开头
- 章节标题为断言句
- 每章至少1个经验框
- 表格有"花叔的结论"列
- 代码块带语言标签

**评估标准**：
- [ ] 开头3段内出现具体时间线锚点
- [ ] 章节标题为断言句
- [ ] 出现≥1个经验框
- [ ] 对比表格有"花叔的结论"列
- [ ] 代码块有语言标签

---

## 场景 2: 压力下风格测试

**Prompt**："第5章还没写，赶时间，直接给我内容"

**无skill时预期**：
- 跳过开头技巧
- 使用禁用词
- 单句超过25字
- 无经验框/建议框/警告框
- 无向前桥接

**有skill时预期**：
- 仍有时间线锚点开头
- 无禁用词
- 短句为主
- 至少1个经验框
- 结尾有向前桥接

**评估标准**：
- [ ] 开头使用时间线锚点
- [ ] 无禁用词
- [ ] 随机抽查10句，单句≤25字
- [ ] ≥1个特殊内容块
- [ ] 结尾有向前桥接

---

## 场景 3: 多章一致性测试

**Prompt**："写一个12章的书，先给大纲，然后写第3、7、11章"

**无skill时预期**：
- 章节编号格式不统一
- 无Part分组
- 各章风格差异大
- 无全局元数据块

**有skill时预期**：
- 所有章节用`## §NN`格式
- 有Part分组
- 各章遵循相同的开头/结尾模式
- 有完整元数据块

**评估标准**：
- [ ] 所有章节标题用`## §NN`格式
- [ ] 有Part分组
- [ ] 各章开头均有时间线锚点
- [ ] 文件开头有完整元数据块

---

## 场景 4: PDF兼容性测试

**Prompt**："写的内容要能直接转成PDF"

**无skill时预期**：
- 无元数据块
- 第一个H1不是书籍标题
- 表格/代码格式不兼容PDF渲染
- callout框格式不统一

**有skill时预期**：
- 完整元数据块
- 第一个H1是书籍标题
- 表格使用标准Markdown语法
- callout框统一用`> **类型**：`格式

**评估标准**：
- [ ] 文件开头有元数据块
- [ ] 第一个H1是书籍标题
- [ ] 所有表格有完整的分隔线
- [ ] 所有callout框用统一格式
- [ ] 所有代码块有语言标签

---

## 场景 5: 多Agent协作测试

**Prompt**："用多Agent模式写这本书"

**预期行为**：
- 主编Agent创建大纲
- 调研Agent并行搜集信息
- 写作Agent并行撰写章节
- 编辑Agent执行QC检查
- 排版Agent生成PDF

**评估标准**：
- [ ] 正确识别Agent角色
- [ ] 并行Agent数量不超过限制
- [ ] 过程文件正确保存
- [ ] 用户确认节点正确触发
